# .gitignore — что НЕ коммитить

**Боль.** `git add .` без разбора утащит в репозиторий мусор: тысячи
файлов `.venv`, локальную базу `db.sqlite3`, кэш `__pycache__/`, а в реальных
проектах — ещё и секреты. Git должен следить только за исходным кодом.
`.gitignore` (*ignore* — «игнорировать») — список того, что git не замечает.

**Когда использую.** Сразу после создания проекта и до первого коммита.

**Минимум содержимого.**

```gitignore
.venv/
__pycache__/
*.py[cod]
db.sqlite3
.env
```

Можно взять готовый шаблон GitHub для Python (`github/gitignore`).

**Как надо.**
- Лучшие практики: заводи `.gitignore` до первого `git add .`; бери готовый шаблон
  GitHub для Python; после пуша проверь, что `.venv` и `db.sqlite3` туда не попали —
  значит, игнор отработал.
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: делать `git add .` без `.gitignore` (утащишь мусор и секреты);
  заводить игнор уже после первого коммита.
- Мои грабли: думал, `.gitignore` удаляет файлы — нет, он лишь прячет их от git, на
  диске они остаются и работают. Критично: секреты (`.env`) — в игнор всегда.
