# GitHub — облачный дом для репозиториев

**Боль.** Проект только на твоём компьютере: умрёт диск — пропадёт всё.
И код негде показать наставнику, не из чего собрать портфолио, нельзя работать с
другого устройства. GitHub — резервная копия, витрина проекта и точка совместной
работы.

**Когда использую.** Когда хочу сохранить работу в облаке, показать её или
открыть с другого компьютера.

**Команды-минимум.**
- `git branch -M main` — назвать главную ветку `main` (`-M` — *move*,
  переименовать).
- `git remote add origin https://github.com/ТВОЙ_ЛОГИН/python-trainer.git` —
  запомнить адрес репозитория под именем `origin` («источник»).
- `git push -u origin main` — отправить и запомнить связь; дальше хватит
  `git push`.

**Как надо.**
- Лучшие практики: поставь GitHub CLI (`gh`, *Command Line Interface*) и сделай
  `gh auth login` один раз — окно входа больше не дёргает; после пуша глянь
  репозиторий в вебе (нет `.venv` и `db.sqlite3` — значит, `.gitignore` сработал).
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: держать код только локально, без бэкапа; при создании репозитория
  добавлять README/.gitignore, когда они уже есть локально (получишь конфликт).
- Мои грабли: создал репозиторий на GitHub с README/.gitignore, а они уже были
  локально — конфликт; пробовал войти обычным паролем — не принят (нужен токен или
  вход через браузер: Git Credential Manager либо `gh auth login`). Критично: в
  публичный репозиторий не должны уехать `.venv`, `db.sqlite3` и секреты.
