# Django admin — готовая панель управления данными

**Боль.** Данными должен управлять и не-программист (контент-редактор): добавлять,
править, удалять записи мышкой. Писать такую CRUD-панель руками — долго и не по делу.

**Когда использую.** Когда нужно человекочитаемо управлять записями модели без кода.

**Команды-минимум.**
- `admin.site.register(Quote)` в `admin.py` — показать модель в админке.
- `python manage.py createsuperuser` — *create superuser*, завести администратора.
- `/admin/` — войти и управлять данными в браузере.

**Как надо.**
- Лучшие практики: регистрируй модель сразу, как завёл; держи `__str__` читаемым,
  чтобы список не пестрил `object (1)`; для удобства — `list_display` с нужными
  колонками.
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: раздавать права суперпользователя кому попало; хранить пароль
  администратора в коде или в git.
- Мои грабли: модель не появлялась в админке, пока не зарегистрировал её в
  `admin.py`; список был нечитаем без `__str__`. Критично: настоящий пароль
  суперпользователя — не в репозитории.
