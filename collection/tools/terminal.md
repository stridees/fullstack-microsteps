# Терминал и навигация по папкам

**Боль.** Мышкой команды разработчику не отдашь: нужно быстро, точно и
воспроизводимо. Терминал — окно, где компьютеру отдают команды текстом; команду
можно скопировать, повторить и дать коллеге.

**Когда использую.** Всегда, когда нужно что-то запустить, поставить или
проверить: версия Python, установка пакета, запуск сервера, любая команда git.

**Команды-минимум.** Это сокращения английских слов:
- `pwd` — *print working directory*, «где я сейчас».
- `ls` — *list*, «что в этой папке».
- `cd projects` — *change directory*, «зайти в папку»; `cd ..` — на уровень вверх.
- `mkdir projects` — *make directory*, «создать папку».
- `~` — короткое имя домашней папки, чтобы не писать полный путь.

**Как надо.**
- Лучшие практики: сверяйся `pwd` перед действием; на macOS/Linux обычно `python3`,
  на Windows бывает удобен лаунчер `py`.
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: запускать команды наугад, не зная, в какой ты папке; держать проект
  вперемешку с чем попало на рабочем столе — потом сам не найдёшь.
- Мои грабли: запускал команды не из той папки, и инструменты путались, где границы
  проекта (спасает привычка сперва `pwd`). Критично: на Windows без галочки
  «Add Python to PATH» команда `python` не найдётся (PATH — список папок, где
  система ищет команды).
