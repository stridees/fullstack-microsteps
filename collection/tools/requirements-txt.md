# requirements.txt — список зависимостей

**Боль.** Папку `.venv` в git не кладут. Тогда как товарищу (или тебе
на другом компьютере) узнать, что именно ставить? «Помню, ставил Django… и что-то
ещё?» — так не пойдёт. `requirements.txt` (*requirements* — «требования,
зависимости») — список пакетов с версиями, по которому окружение воссоздаётся
одной командой.

**Когда использую.** Сразу после установки или обновления зависимостей — чтобы
снимок окружения не отставал от реальности.

**Команды-минимум.**
- `python -m pip freeze > requirements.txt` — *freeze*, «заморозить» список;
  `>` перенаправляет вывод в файл.
- `python -m pip install -r requirements.txt` — `-r` (*requirements*), прочитать
  список из файла; в свежем venv воссоздаёт окружение.

**Как надо.**
- Лучшие практики: этот файл, наоборот, коммить (в отличие от `.venv`) — он честный
  ответ на «что нужно поставить»; воссоздавай окружение в свежем, только что
  активированном venv, чтобы в список не попало лишнего от других проектов.
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: держать зависимости «в голове»; коммитить саму папку `.venv` вместо
  списка.
- Мои грабли: поставил новый пакет и забыл пересоздать `requirements.txt` — у
  товарища проект не завёлся (обновил зависимости — сразу `freeze`). Критично: делай
  `freeze` именно из своего venv, не из системного Python.
