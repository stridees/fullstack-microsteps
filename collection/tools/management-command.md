# Команда управления — своя `manage.py …`

**Боль.** Есть повторяемое действие над данными (залить из файла, почистить,
пересчитать) — а делать его каждый раз руками в `shell` долго и невоспроизводимо.

**Когда использую.** Когда обслуживание данных нужно запускать из терминала
повторяемо — своей командой рядом с `migrate` и `runserver`.

**Команды-минимум.**
- Структура: `<app>/management/commands/<имя>.py` (оба `__init__.py` — пустые).
- Внутри: `class Command(BaseCommand)` с методом `handle(self, *args, **options)`.
- Запуск: `python manage.py <имя>` (например, `loadquotes`).

**Как надо.**
- Лучшие практики: делай команду идемпотентной (`get_or_create`, чтобы повтор не
  плодил дубли); давай понятный вывод (`self.stdout.write(self.style.SUCCESS(...))`);
  заполняй `help` — описание команды.
- Личные находки: *— фиксируй всё, что относится к личному опыту.*

**Как НЕ надо.**
- Антипаттерны: одноразовый скрипт, который при повторном запуске плодит дубли;
  логика загрузки, размазанная по `shell` без следа.
- Мои грабли: команда «не находилась», пока не положил оба пустых `__init__.py` в
  `management/` и `commands/`. Критично: без этих `__init__.py` Django пакет команд
  не видит.
